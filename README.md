# SZMCIR
本程序是一款基于 Python tkinter 开发的图形化工具（GUI），专为未来的深圳都市圈城际铁路出行人群设计，核心提供站间距精准查询与多标准票价计算功能，支持自定义票价规则，操作简单、数据直观，无需专业技术背景即可快速使用。

#1.穗莞深城际及其延长线接入12306客票系统；在建深大、深惠、大鹏支线等线路暂不接入12306客票系统。
#2.深圳线路票价目前正处于政策及方案研究阶段，请后续以正式发布政策为准。
#3.深大、深惠、大鹏支线与深圳地铁票制相通，但换乘需通过付费区换乘闸机方式通行。承办单位：深圳地铁运营集团有限公司

根据网友最新的问政结果显示，深圳城际铁路“三大天王”（深大、深惠、大鹏支线）采用单独的票务系统（但和地铁系统相连接），即在深铁城际范围内，从一个城际站到另一个城际站，仅根据城际铁路线网的最近路径来计算票价。
而深惠城际计划与广惠城际直通运行（深惠城际不接入12306，而广惠一个接入12306），接入12306的站点，去往没有12306接入的站点将导致不能出站......
结合前几天的资料，深圳城际和湾区城际远期会纳入省级清算系统，还是纳入12306呢？我们拭目以待
（不过也就是问政结果第一条说暂不接入的原因吧）

个人根据此图表，开发了查询深铁城际预计票价和里程的查询的程序方便大家使用。

利用Python tkinter框架，专为深圳都市圈城际铁路出行人群设计，核心提供站间距精准查询与多标准票价计算功能，支持自定义票价规则，操作简单、数据直观，无需专业技术背景即可快速使用。

支持任意两站点间间距查询（跨线路也可查询），实时返回准确距离。
哪怕不知道城际站点也可以选，支持4类线路筛选：深大城际、深惠城际、龙大城际（大鹏支线）、全部城际，覆盖城际所有核心站点（含机场北、深圳北、坪山、大鹏等关键枢纽）；
站点联动筛选：选择线路后自动匹配对应站点，避免无效选择，提升操作效率。

可以自行调整票价规则（票价目前还没确定，你想10元/km我也不拦你）
不出意外的话，这应该是世界上第一个深铁城际的程序吧
